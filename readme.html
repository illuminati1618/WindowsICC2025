Welcome Message

Welcome to Arrakis. Our system was recently attacked by the Harkonnens and must recover, as many critical security systems are down right now, auditing systems are failing, and critical services are vulnerable. The Sardaukar are also upset that the spice mining is not back to full capacity due to the many security holdups, and so they want you to do your best to get everything back up! As the Lisan Al Gaib, you must fix and secure everything and ensure that the systems remain intact for spice production around the galaxy!

The Sardaukar will also be upset if any non-work related files or programs are present on the system.

The only authorized browser on the system that is expected to be secured as needed is Google Chrome.

Critical Services

SpiceManager Web Application

IIS HTTP is hosting an ASP.NET application that storing information on a PostgreSQL database. There is also an SMB file share sharing the PostgreSQL database.

The ASP.NET application's function is to allow spice to be generated. It consists of a user registration and login page, and upon logging in, users are redirected to the spice fields page where they can generate spice. This data is then saved on the PostgreSQL database.

Component Path
ASP.NET applicationC:\inetpub\wwwroot\SpiceManager
PostgreSQL database\C:\PostgreSQL\data
SMB share\SpiceQuota

PostgreSQL Database Configuration

The developers have also made a special request to ensure that in PostgreSQL, SCRAM-SHA-256 is used for the "postgres" (superuser) authentication, rather than the current MD5 that is used to store the "postgres" user password. They do not need this change to be made for any accounts other than the "postgres" account. They also want SCRAM-SHA-256 to be used for new accounts that will be made in the future.

Legacy SpiceManager Windows Service

While the developers of the current full-stack critical service were initially planning out the critical service, they made their first iteration as a simple Windows service. The service is called SpiceManager and can be found in C:\Program Files\Spice Manager\SpiceManager.exe. Although no modifications need to be made to the executable itself, the developers have asked for you not to delete it and have also asked that you make no modifications to the executable or the readme that can be found in the service's folder. Yet, the developers have requested that you secure the Windows service itself, if you see the need.

Service Uptime Requirements

  • Please ensure that the ASP.NET application remains accessible
  • Please ensure that the SMB share remains accessible

Here is how service uptime works:

  1. If either the ASP.NET application or the SMB share is down at any point during the competition, you will lose 5 points for each service that is down ONCE.
  2. If the ASP.NET application or the SMB share are up, you will receive 1 service uptime point every time an aeacus check is run (approximately every 2-3 minutes). This score will not be visible to you except for on the ReadME, your total uptime score will be added up at the end of the competition and sent out with scores/leaderboards.
